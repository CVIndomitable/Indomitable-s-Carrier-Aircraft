# 不挠的舰载机 - 子系统划分

## 项目概述

基于皮兰港现有基础设施，扩展并重构舰载机系统，实现完整的 AI 控制、编组管理、火控系统和控制面板。

## 现有基础设施（可复用）

## 新增子系统划分

### 1. 飞机实体系统（增强）

**包路径**: `com.piranport.aircraft.entity`

**职责**: 扩展现有 `AircraftEntity`，实现双武器槽和数据驱动机型

#### 核心类

```
AircraftEntity (扩展现有)
├── 双武器槽系统
│   ├── airWeaponSlot: WeaponSlot   // 对空武器槽（弹匣）
│   └── seaWeaponSlot: WeaponSlot   // 对海武器槽（炸弹/鱼雷/火箭）
├── 机型配置接口
│   ├── AircraftConfig (新建)
│   │   ├── 速度、HP、载弹量
│   │   ├── 对空能力类型（前射/自卫炮塔/两者/无）
│   │   ├── 对海能力标志（俯冲/水平/鱼雷/火箭）
│   │   ├── 自卫球形范围
│   │   └── 探测范围（反潜机）
│   └── AircraftConfigRegistry (新建)
└── 飞行物理增强
    ├── 俯冲拉起算法
    ├── 航向平滑转向
    └── 盘旋算法优化
```

**现有基础**: `AircraftEntity` 已有基础飞行和状态机，需要扩展为数据驱动。

**新增内容**:
- 双武器槽管理
- JSON 配置加载（机型参数完全由数据驱动）
- 俯冲轰炸的飞行物理
- 盘旋算法优化

### 2. 状态机系统（重构）

**包路径**: `com.piranport.aircraft.state`

**职责**: 重构现有状态机，支持方案中的完整状态流转

#### 核心接口

```
IPlaneState (新建)
├── onEnter(PlaneEntity plane)
├── onTick(PlaneEntity plane)
└── onExit(PlaneEntity plane)

PlaneStateManager (新建)
└── 状态切换协调器
```

#### 状态实现类

```
states/
├── StandbyState         // 玩家头顶待命（扩展现有 LAUNCHING）
├── OrbitingState        // 盘旋点待命（新建）
├── ApproachState        // 赶赴坐标（扩展现有 CRUISING）
├── CirclingState        // 通用盘旋（新建）
├── LockedState          // 锁定目标，计算三坐标（新建）
├── AttackingState       // 攻击航段（扩展现有 ATTACKING）
├── DroppingState        // 投弹动作（新建，从 ATTACKING 拆分）
├── PostAttackState      // 确认打击效果（新建）
├── DogfightState        // 空战追击（新建）
└── ReturningState       // 返航（扩展现有 RETURNING）
```

**现有基础**: `AircraftEntity.FlightState` 枚举（6个状态）

**重构方案**:
- 保留现有枚举作为网络同步标记
- 新增接口模式实现复杂状态逻辑
- 状态类可组合复用（如多种状态都需要盘旋）

### 3. 弹道系统（增强）

**包路径**: `com.piranport.aircraft.ballistic`

**职责**: 扩展现有弹道计算，实现三坐标体系和各弹药类型

#### 核心类

```
ThreeCoordinateCalculator (新建)
├── calculateImpactPoint(target, weapon)    // 目标坐标 -> 弹着点
├── calculateDropPoint(impact, aircraft)    // 弹着点 -> 投弹坐标
└── calculateTurnPoint(drop, aircraft)      // 投弹坐标 -> 转向点

弹药弹道类型（扩展现有）
├── FreeFallBallistic        // 自由落体（水平轰炸）- 扩展 AerialBombEntity
├── GuidedBallistic          // 制导（俯冲轰炸）- 扩展 MissileEntity
├── TorpedoBallistic         // 鱼雷（线性外推）- 扩展 TorpedoEntity
└── RocketBallistic          // 火箭（直线）- 新建或扩展 MissileEntity
```

**现有基础**: 
- `BallisticSolver` - 弹道计算
- `AerialBombEntity` - 炸弹实体
- `TorpedoEntity` + `TorpedoGuidanceManager` - 鱼雷系统
- `MissileEntity` - 导弹实体

**新增内容**:
- 三坐标计算封装
- 各弹药类型的飞行时间参数配置
- 命中偏差随机化（水平轰炸）
- 制导丢失概率（俯冲轰炸）

### 4. 锁定子系统（新建）

**包路径**: `com.piranport.aircraft.targeting`

**职责**: 独立于状态机的目标管理，优先级判断

#### 核心类

```
TargetingSubsystem (新建)
├── resolveTarget(aircraft): Entity
│   ├── 优先级1: 检查火控指定目标
│   └── 优先级2: 自动锁定档位
├── AutoLockMode (枚举)
│   ├── NEAREST      // 最近目标
│   ├── STRONGEST    // 威胁值最高
│   ├── FOCUS        // 集火
│   ├── SPREAD       // 分散
│   └── TYPE_FILTER  // 类型优先
└── ThreatEvaluator (新建)
    └── calculateThreat(entity): float
```

**现有基础**: 
- `FireControlManager` - 火控目标列表
- `AircraftAttackMode` - FOCUS/SPREAD/FOLLOW 模式

**整合方案**:
- `FireControlManager` 保持不变（提供火控目标列表）
- `TargetingSubsystem` 封装优先级逻辑
- `AircraftAttackMode` 映射到 `AutoLockMode`

### 6. 编组系统（新建）

**包路径**: `com.piranport.aircraft.formation`

**职责**: 三级编组管理、等齐出发、任务过滤

#### 核心类

```
FormationManager (新建)
├── Squadron (中队) - 核心单位
│   ├── UUID squadronId
│   ├── SquadronType (对空/鱼雷/轰炸/反潜)
│   ├── List<UUID> aircraftIds
│   └── attackCapability: EnumSet<AttackType>
├── Flight (小队) - 同机型子单位
│   └── 归属于 Squadron
└── Wing (大队) - 混编调度单位
    ├── List<Squadron> squadrons
    ├── AirDefenseMode (一般/主动/低攻击性)
    └── Vec3 rallyPoint (盘旋点)

FormationCoordinator (新建)
├── waitForAssembly(Formation): boolean  // 等齐出发
├── filterByTask(Wing, TaskType): List<Squadron>  // 任务过滤
└── manageAirDefense(Wing): void  // 对空模式控制
```

**数据结构**:
```
玩家 UUID -> Wing (每个玩家一个大队)
  Wing -> List<Squadron>
    Squadron -> List<Flight>
      Flight -> List<Aircraft UUID>
```

**持久化**: 使用 Player Data Component 或 SavedData

## 数据驱动配置

### 机型配置文件（JSON）

```json
// data/piranport/aircraft_config/b25.json
{
  "type": "b25",
  "display_name": "B-25 米切尔",
  "speed": 1.2,
  "health": 100.0,
  "air_weapon": {
    "type": "both",  // front_gun | turret | both | none
    "front_gun_damage": 5.0,
    "turret_damage": 3.0,
    "turret_range": 16.0
  },
  "sea_weapon": {
    "allowed": ["bomb"],  // bomb | torpedo | rocket
    "attack_mode": "level_bombing",  // level_bombing | dive_bombing
    "capacity": 8
  },
  "flight_params": {
    "turn_radius": 12.0,
    "dive_angle": 0,  // 不支持俯冲
    "pull_up_distance": 20.0
  }
}
```

### 弹药配置文件（JSON）

```json
// data/piranport/ammo_config/aerial_bomb_500kg.json
{
  "type": "aerial_bomb_500kg",
  "display_name": "500kg 航空炸弹",
  "damage": 50.0,
  "blast_radius": 8.0,
  "ballistic_type": "free_fall",
  "flight_time_coefficient": 1.0,  // 用于计算弹着点
  "deviation": {
    "horizontal": 2.0,  // 水平偏差（方块）
    "vertical": 0.5
  }
}
```

## 开发优先级

### Phase 1: 核心系统（6-8周）

1. **飞机实体双武器槽** (1周)
   - 扩展 `AircraftEntity`
   - 双武器槽数据结构
   - 网络同步

2. **状态机重构** (2周)
   - `IPlaneState` 接口
   - 基础状态实现（STANDBY, APPROACH, ATTACKING, RETURNING）
   - 状态切换逻辑

3. **三坐标弹道系统** (2周)
   - `ThreeCoordinateCalculator`
   - 自由落体弹道（水平轰炸）
   - 基础命中测试

4. **锁定子系统** (1周)
   - `TargetingSubsystem`
   - 与现有 `FireControlManager` 对接
   - NEAREST/FOCUS 模式

5. **基础测试** (1-2周)
   - 单机型水平轰炸完整流程
   - 火控锁定 -> 飞行 -> 投弹 -> 返航

### Phase 2: 多机型与弹道（4-6周）

6. **机型数据驱动** (2周)
   - JSON 配置加载
   - `AircraftConfigRegistry`
   - 3-4个机型实现

7. **俯冲轰炸** (1周)
   - 制导弹道
   - 俯冲拉起飞行物理

8. **鱼雷攻击** (1周)
   - 鱼雷弹道（扩展现有）
   - 鱼雷投放流程

9. **前射型空战** (1-2周)
   - `DogfightState`
   - 追击算法
   - 自卫炮塔模式

### Phase 3: 编组与控制面板（6-8周）

10. **编组系统** (3周)
    - `FormationManager`
    - 三级结构（Wing/Squadron/Flight）
    - 等齐出发逻辑
    - 任务过滤

11. **控制面板 GUI（简化版）** (3-4周)
    - 终端物品
    - 配置面板（无地图）
    - 指挥面板
    - 弹药装载管理
    - 网络包通信

12. **目标分配** (1周)
    - `TargetAssignmentManager`
    - 集火/普通模式

### Phase 4: 高级功能（4-6周）

    - `LeaderSelectionManager`
    - 多长机切换
    - 长机击落转移

14. **控制面板地图** (2-3周)
    - 小地图渲染

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 三坐标计算不准确 | 高 | 先做简化版（固定高度），逐步迭代；大量测试 |
| 状态机过于复杂 | 中 | 接口模式解耦，单元测试每个状态 |
| 编组系统性能问题 | 中 | 限制同时运行飞机数量；目标搜索缓存 |
| GUI 地图渲染复杂 | 高 | Phase 1 不做地图，Phase 3 只做静态标记 |
| 多人同步问题 | 中 | 复用现有网络框架；充分测试多人环境 |
| 区块加载泄漏 | 高 | 严格追踪长机生命周期；服务器关闭时清理 |
